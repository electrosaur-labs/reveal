## Recent Session Summary (2026-02-27)

### Palette Override Stability Fix ✅ COMPLETE (`e50377a`)
Palette overrides (user-chosen ink colors) were disappearing after being set.

**Root cause:** `ProxyEngine.updateProxy()` applied palette override BEFORE knobs. `MechanicalKnobs.applyMinVolume()` classifies colors into 12 hue sectors for rescue decisions — the overridden color's new hue sector could lose its sector rescue, causing the swatch to be pruned and disappear.

**Fix:** Reorder `updateProxy()` to apply knobs first (on baseline palette), then palette override. Also added `_scoringGeneration++` to all 6 palette surgery methods in SessionState to cancel stale background ΔE scoring.

**Palette override preservation across re-posterization:** When a structural parameter change triggers re-posterization, user overrides are now saved and re-injected into the closest new palette slots by Lab distance.

### Metadata Persistence Fix ✅ COMPLETE (`e50377a`)
XMP metadata (separation manifest) was silently not persisting in the PSD.

**Root cause:** Metadata writing was inside `suspendHistory` block (moved there in `d4fdf35`). UXP batchPlay `set fileInfo` inside `suspendHistory` silently drops the write.

**Fix:** Moved metadata writing back to a **separate `executeAsModal`** from layer creation.

### Structured reveal: XMP Namespace ✅ COMPLETE (`a3a97e0`)
Replaced JSON blob in `dc:description` with human-readable caption. Structured machine-readable data now lives in a custom `reveal:` XMP namespace.

**Key discovery:** `set XMPMetadataAsUTF8` batchPlay works fine — the previous failures were from `XMPMeta.serialize()` mangling packet padding. Raw string injection (read existing XMP, splice in XML, write back) bypasses the broken serializer.

- `writeManifestXMP` renamed to `writeManifestIPTC` (human-readable IPTC fields)
- `writeStructuredXMP` rewritten with raw string injection (no XMPMeta dependency)
- Version fields clarified: `reveal:version` = plugin version (1.0.0), `reveal:dnaVersion` = DNA schema version (2.0)
- `<!-- REVEAL:BEGIN -->` / `<!-- REVEAL:END -->` sentinels for idempotent re-writes

## Parameter Category Architecture

Parameters are classified into mutually exclusive categories in SessionState:

| Category | Set | Behavior | Examples |
|----------|-----|----------|----------|
| **MECHANICAL_KNOBS** | 3 params | Fast path — mask re-render only | minVolume, speckleRescue, shadowClamp |
| **PRODUCTION_KNOBS** | 2 params | Production render only | trapSize, meshSize |
| **STRUCTURAL_PARAMS** | 25 params | Slow path — full re-posterize | targetColors, engineType, distanceMetric, ... |
| **UNIMPLEMENTED_KNOBS** | 5 params | UI exists, no engine reads | vibrancyThreshold, detailRescue, medianPass, maskProfile, neutralCentroidClampThreshold |

---

## Key Design Decisions

### XMP Metadata: Separate Modal Required (2026-02-27)
`set fileInfo` inside `suspendHistory` silently drops. Metadata writes MUST be in a separate `executeAsModal` from layer creation.

### XMP Structured Write: Raw String Injection (2026-02-27)
`XMPMeta.serialize()` mangles padding → Photoshop rejects. Read existing XMP as string, splice in `reveal:` namespace XML, write back. `set XMPMetadataAsUTF8` batchPlay works fine with hand-crafted XML.

### Palette Override: Knobs Before Override (2026-02-27)
ProxyEngine must apply knobs BEFORE palette override. minVolume's hue sector classification uses palette Lab values — if override changes the hue, sector rescue may fail and the swatch disappears.

### meshSize is Session-Level Equipment (2026-02-26)
Physical property of printing equipment, not archetype-driven. Default 230 TPI, survives archetype swaps.

### Production Render Uses Locked Palette (No Re-Posterize)
Palette curated via Surgery Mode. Re-posterizing would invalidate overrides.
