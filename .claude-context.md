## Active Bug: Palette Surgery Swatch Corruption

**Status:** DEBUGGING — root cause not yet identified

**Symptoms (user-reported):**
1. All swatches acquire undo badges (override dots) after interacting
2. All swatches become the same color as the changed swatch
3. These happen "as I click on them" — not just after merge/edit

**What we've tried:**
- Rewrote PaletteSurgeon from scratch using Architect's skeleton (simpler 2-state machine)
- Verified `_buildOverriddenPalette()` is read-only (no side-effects)
- Verified `mergePaletteColors` reads from baseline+overrides
- Verified ProxyEngine._applyMinVolume preserves index stability
- Verified CSS uses `display:none` by default, `.visible` to show override dot/revert
- Verified `rgbToLab` / `labToRgb` return correct `{L,a,b}` / `{r,g,b}` formats

**What we've ruled out:**
- CSS visibility issue (dots are `display:none` by default)
- Format mismatch (Lab/RGB conversions are correct)
- Baseline contamination (_snapshotBaseline only called in initializeProxy/rePosterize)
- Direct paletteOverrides mutation in _buildOverriddenPalette (it's pure read)

**Leading hypothesis (needs verification):**
- Async reentrancy: `overridePaletteColor` and debounced `triggerProxyUpdate` may interleave, both calling `updateProxy` concurrently. Since `updateProxy` does `_restoreFromBaseline()` then `await _applyKnobs()`, a second call during the await could replace separationState mid-operation.
- OR: Something we haven't traced is adding entries to paletteOverrides

**Next step:** Add diagnostic logging to `_rebuild`, `mergePaletteColors`, `overridePaletteColor`, and `_buildOverriddenPalette` so we can see actual runtime state in UXP console.

## Key Design Decisions

### Palette Override Pipeline (2026-02-14)
Order in `updateProxy`: Restore baseline → Apply palette override → Apply knobs. Overrides indexed against baseline. `_buildOverriddenPalette` reads from `_baselineState.palette`.

### MinVolume Preserves Index Stability (2026-02-14)
Weak colors remapped in colorIndices, palette array length unchanged. Override indices stay valid.

### Production Render Uses Locked Palette (No Re-Posterize)
Palette curated via Surgery Mode. Re-posterizing would invalidate overrides.

### Proxy Preview Uses Mask-Based Rendering
Preview from masks (not raw colorIndices) so speckleRescue effects are visible.
