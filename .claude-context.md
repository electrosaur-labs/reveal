## Next Phase: Dynamic Hue Anchoring

**Status:** 🏗️ Architecture defined, not yet implemented
**Document:** `DYNAMIC-HUE-ANCHORING-ARCHITECTURE.md`

**Goal:** Replace hardcoded hue penalties with DNA-derived anchors

**Current (Hardcoded):**
```javascript
// PosterizationEngine.js - binary on/off switch
if (hue >= 70 && hue <= 95) {
    distance *= 256; // All yellows treated the same
}
```

**Proposed (DNA-Derived):**
```javascript
// Use image's actual yellow centroid as anchor
const anchor = dna.sectors.yellow.hMean; // e.g., 72° for ochre, 100° for lemon
const drift = Math.abs(pixelHue - anchor);
const huePenalty = Math.pow(drift, 2) * config.hueLockSensitivity; // Gaussian curve
```

**Benefits:**
- Adapts to each image's actual yellows
- Smooth falloff instead of binary gate
- Self-correcting for underexposed/overexposed images
- Prevents cross-sector "bullying" (orange dominating yellow)

**Implementation Location:**
- `packages/reveal-core/lib/engines/PosterizationEngine.js:_calculateColorDistance()`
