## Parameter Category Architecture

Parameters are classified into mutually exclusive categories in SessionState:

| Category | Set | Behavior | Examples |
|----------|-----|----------|----------|
| **MECHANICAL_KNOBS** | 3 params | Fast path вЂ” mask re-render only (no re-posterize) | minVolume, speckleRescue, shadowClamp |
| **PRODUCTION_KNOBS** | 2 params | Production render only вЂ” no proxy update | trapSize, meshSize |
| **STRUCTURAL_PARAMS** | 25 params | Slow path вЂ” full re-posterize of 512px proxy | targetColors, engineType, distanceMetric, lWeight, cWeight, ... |
| **UNIMPLEMENTED_KNOBS** | 5 params | UI exists but no engine reads them вЂ” no re-posterize | vibrancyThreshold, detailRescue, medianPass, maskProfile, neutralCentroidClampThreshold |

`ALL_KNOBS` = union of all four sets. Used for config sync, dirty detection, reset.

**Contract test:** `test/unit/parameter-flow-contract.test.js` enforces no param is in multiple categories and every `generate()` output key maps to an engine or documented unimplemented set.

## Key Design Decisions

### meshSize is Session-Level Equipment (2026-02-26)
Screen mesh TPI is a physical property of the printing equipment, not an image/archetype property. Default 230 TPI, survives archetype swaps, only affects production render (not proxy). Maps to `meshCount` at the SeparationEngine boundary.

### chromaGate Recalculation (2026-02-26)
`_baseCWeight` stored when archetype loads (config.cWeight / config.chromaGate). `_rebuildConfigFromState()` recalculates `cWeight = _baseCWeight Г— chromaGate`. Manual cWeight changes update the base.

### UXP Keyboard Limitation (2026-02-23)
UXP non-modal dialogs do NOT receive keyboard events. All interaction must use pointer events.

### Posterize-on-Demand (2026-02-22)
Top 8 archetype cards get eagerly О”E-scored and posterized. Rest posterize lazily when clicked.

### Production Render Uses Locked Palette (No Re-Posterize)
Palette curated via Surgery Mode. Re-posterizing would invalidate overrides.

### Proxy Preview Uses Mask-Based Rendering
Preview from masks (not raw colorIndices) so speckleRescue effects are visible.
